# Contributing to Vulhub

Thank you for your interest in contributing to Vulhub!

The complete guide for adding a new vulnerability environment — including project structure, naming conventions, Dockerfile guidelines, docker-compose.yml format, documentation style, CI checks, and a pre-submission checklist — is maintained in [SKILL.md](.claude/skills/add-vulhub-env/SKILL.md). Please read it before starting your contribution.
